Engram · memoria persistente

Cómo usar Engram en este repositorio

Engram guarda observaciones importantes sobre decisiones, bugs y contexto de la sesión para que futuros agentes recuperen memoria precisa y útil.

¿Qué guarda Engram?

Siempre que se completa una corrección de bug, se toma una decisión arquitectónica, se descubre un comportamiento no obvio o se cambia la configuración, se ejecuta mem_save para registrar:

  • Qué se hizo y por qué.
  • En qué archivos sucedió.
  • Qué aprendizajes o gotchas importantes surgieron.

Así la próxima vez el agente ya tiene contexto y puede continuar desde donde quedó.

Cuando guardar

Ejecuta mem_save al terminar:

  • Correcciones de bugs.
  • Decisiones de diseño o arquitectura.
  • Cambios de configuración o instrucciones nuevas del usuario.
  • Descubrimientos técnicos relevantes.

Cuando buscar

Antes de empezar o cuando el usuario solicita recuerdo:

  • Usa mem_context para obtener sesiones recientes.
  • Si no ves lo que necesitas, busca por tema con mem_search.
  • Relee observaciones completas con mem_get_observation.

Siempre que termines

Cierra la sesión con mem_session_summary incluyendo:

  • Meta general.
  • Instrucciones o restricciones aprendidas.
  • Descubrimientos técnicos.
  • Trabajos completados y próximos pasos.

Flujo típico

  1. Inicia sesión y revisa mem_context para saber qué se hizo antes.
  2. Trabaja y, si haces un cambio significativo, registra con mem_save.
  3. Si hay que recordar algo después, consulta la memoria guardada.
  4. Antes de cerrar, resume y marca archivos relevantes en mem_session_summary.

Ejemplo de observación

Un mem_save bien formado:

**What**: Reemplacé `RuntimeException` por `InvalidOperationException` en `LaminaService`
**Why**: Para devolver 400 con DTO consistente y no exponer detalles internos
**Where**: src/.../LaminaService.java
**Learned**: Los handlers globales esperan excepciones específicas para armar `ApiResponseDTO`

Consejos rápidos

  • No escribas en memoria información superflua (por ejemplo, "corrí tests") si no aporta contexto.
  • Utiliza títulos breves y consistentes al guardar (p.ej. "Decisión · Validación de pedidos").
  • Si corriges un bug, describe el síntoma y cómo lo resolviste.
  • Evita duplicar temas; usa topic_key si el asunto evoluciona.
Engram mantiene el contexto vivo. Guarda, busca y resume cada sesión para que el equipo no pierda la historia.
